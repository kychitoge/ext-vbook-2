# Skill: Create New VBook Extension

**Trigger**: User says "táşˇo extension", "create extension", "add source", or provides a website URL with intent to add it.

## Prerequisites

Before writing any code, run:
```bash
vbook check-env --json
```
â†’ Verify `VBOOK_IP` is set and device is reachable. If not, stop and ask for IP.

---

## Step-by-Step Workflow

### STEP 1 â€” Initialize scaffold

```bash
vbook create "<name>" --source "<url>" --type <novel|comic|chinese_novel>
```

**Rules:**
- `<name>` = short identifier (no spaces), e.g. `truyenfull`, `ntruyen`
- `--source` = root domain only, no trailing slash, e.g. `https://truyenfull.io`
- `--type` = use `novel` for Vietnamese text, `comic` for manga/images, `chinese_novel` for Chinese text
- `--locale vi_VN` (default), `zh_CN` for Chinese sites, `en_US` for English
- Add `--tag nsfw` only if the site has adult content

### STEP 2 â€” Analyze website structure

```bash
vbook analyze "<url>" --json
```

Parse the JSON output to get:
```json
{
  "tech": { "cloudflare": false, "nextjs": false, "gbk": false },
  "listSelectors": { ".truyen-item": 20, ".book-item": 8 },
  "nameLinkSelectors": { "h3.truyen-title a": 20 },
  "coverSelectors": { ".truyen-item img": 20 },
  "paginationSelectors": { ".pagination a": 5 }
}
```

**Decision tree based on `tech`:**
- `cloudflare: true` â†’ All scripts MUST use `Engine.newBrowser()` instead of `fetch()`. If manual captcha is needed, explicitly instruct the user to open the VBook app on their phone and pass the captcha.
- `nextjs: true` or `nuxt: true` â†’ Use `b.launch(url, 10000)` + `b.callJs()` to wait for content
- `gbk: true` â†’ Use `res.html("gbk")` instead of `res.html()`
- All `false` â†’ Standard `fetch(url)` + `response.html()` is fine

### STEP 3 â€” Implement scripts

Navigate to `extensions/<name>/src/` and implement each file.

**Implementation order:** `detail.js` â†’ `toc.js` â†’ `chap.js` â†’ `gen.js` â†’ `home.js` â†’ `search.js`

Use selectors from Step 2. Key rules (NEVER violate):
- **ES6 Limits**: No `async/await`, no `?.`, no `??`, no `...spread` calls, no default params. Use ES5 equivalents.
- **ALL scripts**: MUST export `function execute(...)`.
- **Pagination**: `nextPage` MUST be a string: `String(parseInt(page) + 1)`.
- **Config**: Create `src/config.js` with:
  ```js
  let BASE_URL = "https://...";
  try { if (CONFIG_URL) BASE_URL = CONFIG_URL; } catch(e) {}
  ```
  Use `load('config.js');` in all scripts. MUST use `let` (not `const`) to allow VBook domain injection.
- **URL Normalization**: In `detail`, `toc`, `chap`, normalize the url on first line: `url = url.replace(/^(?:https?:\/\/)?(?:[^@\n]+@)?(?:www\.)?([^:\/\n?]+)/img, BASE_URL);`.
- **Fetch Params**: Use `{ queries: { key: val }}` in `fetch()` instead of string concatenation.
- **APIs First**: For `toc.js`, check if the website has hidden input APIs (`#truyen-id`) or `window.__NUXT__` before scraping HTML pages sequentially.
- **Rich Data**: ALWAYS extract `genres` array and `suggests` array (e.g. author's other books) in `detail.js`.
- **Browser API Option**: Use `Engine.newBrowser()` to intercept API calls or execute JS if the site acts as an SPA/NextJS.

Reference: See `docs/vbook_demo.md` for complete working examples.

### STEP 4 â€” Validate

```bash
vbook validate ./extensions/<name>
```

If errors â†’ Fix all errors before proceeding. Re-run validate until 0 errors.
Warnings are OK to proceed.

### STEP 5 â€” Debug individual scripts

Test `detail.js` first (most critical):
```bash
vbook debug extensions/<name>/src/detail.js -in "<book_url>" --json
```

Parse output:
```json
{
  "success": true,
  "data": { "name": "...", "author": "...", "ongoing": true },
  "log": "...",
  "time": "2.1s"
}
```

If `exception` in response â†’ analyze error message + fix â†’ re-run debug.

Common fixes:
- `Cannot read property of null` â†’ selector returned nothing â†’ update CSS selector
- `ClassCastException` â†’ function named wrong (e.g. `home()`) â†’ rename to `execute()`
- Blank `data` â†’ wrong selector â†’ inspect HTML manually or re-run `vbook analyze`

Then test remaining scripts:
```bash
vbook debug extensions/<name>/src/toc.js -in "<detail_url>" --json
vbook debug extensions/<name>/src/chap.js -in "<chapter_url>" --json
vbook debug extensions/<name>/src/search.js -in "tien hiep" --json
vbook debug extensions/<name>/src/genre.js --json
```

### STEP 6 â€” Full flow test

```bash
vbook test-all --json
```

**Note:** `test-all` only covers `home->gen->detail->toc->chap`. You MUST test `search.js` and `genre.js` manually in STEP 5!

**ALSO:** After `test-all` passes, verify:
- `icon.png` is a valid image file (not HTML)
- Links generated by `search.js` match the real website URL format
- Test with a REAL book URL from the website

If fails at a step â†’ fix that script â†’ re-run with `--from <step>` to save time:
```bash
vbook test-all --from detail --json   # skip home and gen
vbook test-all --from toc --json      # skip home, gen, detail
```

### STEP 7 â€” Publish

```bash
vbook publish
```

This validates â†’ builds `plugin.zip` â†’ updates root `plugin.json` registry.

After publish, report:
- Extension name and version
- URL configured in `plugin.json metadata.source`
- Which scripts were created (home/gen/search/detail/toc/chap)

## Error Recovery Cheatsheet

| Error | Cause | Fix |
|-------|-------|-----|
| `home.js: No data` | Wrong list selector | Re-run `vbook analyze <list_url>` and update `gen.js` selector |
| `detail.js exception: null` | Cover/author selector null | Add null guard: `var cover = el ? el.attr("src") : ""` |
| `toc.js: No chapters` | Chapter list needs JS rendering | Switch to `Engine.newBrowser()` |
| `chap.js empty content` | Wrong content container selector | Try broader: `#content, .chapter-content, [class*=content]` |
| `vbook analyze` fails | Device unreachable | Run `vbook check-env` first |
| GBK garbled text | Chinese site with GBK encoding | Use `res.html("gbk")` everywhere |
